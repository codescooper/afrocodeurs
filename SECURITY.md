# Politique de sécurité

## Versions supportées

Le projet est en développement actif. Les correctifs de sécurité s'appliquent à
la branche par défaut (`main` / `master`).

## Signaler une vulnérabilité

Merci de **ne pas ouvrir d'issue publique** pour une faille de sécurité — cela
exposerait les utilisateur·rice·s avant qu'un correctif n'existe.

Signale-la en privé :

- de préférence via les **GitHub Security Advisories** (onglet _Security_ →
  _Report a vulnerability_), une fois le dépôt sur GitHub ;
- ou par email à **security@afrocodeurs.org**.

Merci d'inclure :

- une description de la vulnérabilité et de son impact,
- les étapes pour la reproduire,
- la version ou le commit concerné.

## Notre engagement

- **Accusé de réception sous 72 h.**
- On te tient informé·e jusqu'à la résolution.
- Merci de nous laisser un délai raisonnable pour corriger **avant toute
  divulgation publique**.
- Pas de programme de bug bounty pour l'instant — mais ta contribution sera
  **créditée** (avec ton accord). 🙏
